Abstract

pvaSrv is a pvAccess server running on top of an EPICS V3 database, implemented in C++.

This product is part of the V4 implementation of EPICS (Experimental Physics and Industrial Control System).

Status of this Document

This is the 23-Jan-2013 version of pvaSrv.

Table of Contents

Introduction

pvaSrv is a pvAccess server that runs in the EPICS V3 IOC.

It allows you to get, put and monitor V3 PVs (fields of V3 records) over pvAccess, translating the value and its meta data (graphics limits, alarm status, timestamp) into Normative Type (NT) pvData structures (NTScalar, NTScalarArray).

Usage

To use pvaSrv in a V3 IOC created in yourApp, you have to do the following steps:

  1. Add the pvaSrv dbd to your IOC, for example in myAppInclude.dbd 
    include "base.dbd"
include "pvaSrv.dbd"
  2. Add the EPICSv4 libraries, for example in myApp/src/Makefile 
    myApp_LIBS += pvaSrv pvIOC pvAccess pvData
myApp_LIBS += $(EPICS_BASE_IOC_LIBS)
  3. Start pvaSrc in your st.cmd, for example in iocBoot/myApp/st.cmd 
    iocInit()

epicsThreadSleep(2.0)
startV3Channel
